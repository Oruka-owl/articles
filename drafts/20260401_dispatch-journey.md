## まず環境を整えるだけで半日かかった

最初の壁は「そもそもGitHubを使ったことがない」というところでした。アカウントを作って、gh CLI（GitHubをターミナルから操作するツール）をインストールして、Homebrewも入れて——と、前提知識ゼロからスタートしたので、「なんでこのコマンドが動かないんだ」という時間が思いのほか長かったです。

Homebrewはmacのパッケージ管理ツールで、開発者ならほぼ全員使っているものなのですが、インストール自体に少し時間がかかります。そこから`brew install gh`でgh CLIを入れて、`gh auth login`でGitHubと連携する。文字にすると3行ですが、初めてだと何をどこに打てばいいのかわからず、調べながら進めたので相当な時間を消費しました。

articlesリポジトリ（`github.com/Oruka-owl/articles`）を作ったのもこのタイミング。「リポジトリ」とはGitHubにファイルをまとめて管理する箱のようなものです。ここが記事の保存場所になります。

## Claude Codeに「好きにやっていいよ」と言う設定

次の関門は、Claude Codeが自律的に動けるようにする設定でした。

通常のClaude Codeは、コマンドを実行するたびに「これを実行していいですか？」と確認を求めてきます。人間が手元で使うぶんには安全でいいのですが、遠隔から指示を送るとなると毎回承認できません。そこで必要なのが**bypassPermissions**の設定です。

`~/.claude/settings.json`に以下を書き加えます。

```json
{
  "bypassPermissions": true
}
```

これでClaude Codeは確認なしで作業を進められるようになります。ただし、これは信頼できる自分の環境だからこそできる設定です。見知らぬリポジトリやサーバーで使うのはリスクがあるので注意が必要です。

## SSHでつないだらキーチェーンに詰まった

環境が整ったところで、本題の「iPhoneからの遠隔操作」に取り組みます。

iPhoneのSSHショートカット（ショートカットアプリ＋SSH接続）でMacに繋いで、Claude Codeを動かす——というのが当初の設計でした。SSHで繋ぐこと自体はできたのですが、ここで想定外の問題が起きました。

**キーチェーンにアクセスできない**のです。

macOSのキーチェーンは、パスワードや認証情報を安全に保管する仕組みですが、SSHでリモートログインした場合、そのキーチェーンにはアクセスできません。Claude Codeの認証情報がキーチェーンに入っていたため、SSH越しに起動しようとするとエラーになってしまいました。

解決策は**CLAUDE_ACCESS_TOKEN環境変数**を使う方法です。Claude Codeの認証トークンを環境変数として直接セットすることで、キーチェーンを経由しなくても認証が通るようになります。

```bash
export CLAUDE_ACCESS_TOKEN="your_token_here"
```

これを`.zshrc`や`.bashrc`に書いておくか、SSH接続時のセッションにセットする形で対処しました。

## tmuxで「常駐するClaude」を作る

次に直面したのは「SSHを切ったらセッションが終わる」問題です。

iPhoneからSSHで繋いでClaude Codeを起動しても、接続を切った瞬間にプロセスが死んでしまいます。「指示だけ送って、あとはバックグラウンドで動いててほしい」という要件には応えられません。

そこで使ったのが**tmux**です。tmuxはターミナルのセッション管理ツールで、SSH接続を切ってもセッションが生き残ります。

```bash
# tmuxセッションを作成して名前をつける
tmux new-session -d -s claude-session

# セッションの中でClaude Codeを起動
tmux send-keys -t claude-session "claude" Enter
```

こうしてMacの上に「常駐するClaude Codeのセッション」を作りました。iPhoneからSSHで接続し、tmuxセッションにアタッチして指示を送り、デタッチして接続を切る。Claudeは裏でせっせと作業を続けてくれます。

## Tailscaleで外出先からも繋がるように

自宅のWi-Fiの中だけなら問題なく動いていたのですが、「外出先からも使いたい」となるとまた話が変わります。自宅のMacはインターネット上に直接さらされているわけではないので、外から繋ぐにはひと工夫必要です。

そこで導入したのが**Tailscale**です。TailscaleはVPNの一種で、自分のデバイス同士をプライベートなネットワークで繋いでくれます。MacとiPhoneの両方にTailscaleを入れてサインインするだけで、外出先でもまるで同じWi-Fiにいるかのように接続できるようになります。

設定もほぼポチポチするだけで、ここは比較的スムーズに進みました。Tailscaleのおかげで「どこにいてもMacに繋げる」環境が完成しました。

## iPhoneのショートカットアプリで完成した指令システム

最後のピースは、iPhoneから指示を送るインターフェースです。

ターミナルアプリを開いてコマンドを打つのでは、結局手間がかかります。そこでiPhoneのショートカットアプリを使って、テキストを入力したら自動でSSH接続してClaude Codeに指示を送る仕組みを作りました。

ショートカットの流れはシンプルです。

1. テキスト入力フォームが開く
2. 入力した内容がSSH経由でMacに送られる
3. tmuxセッション上のClaude Codeがその指示を受け取る
4. Claudeが作業してGitHubにpushする

このシステムが完成した瞬間、試しに「新人看護師向け採血講義資料を作ってGitHubにpushして」と送ってみました。しばらくして確認したら、ちゃんとリポジトリにファイルが追加されていました。

**iPhoneから指示を送って、Macが動いて、GitHubに成果物が届いた。**

こんな小さなことで、なぜか泣きそうになりました。
